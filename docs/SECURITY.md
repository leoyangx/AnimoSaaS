## 报告漏洞

请勿通过 GitHub Issues 公开报告安全漏洞。

发送邮件至项目维护者，或通过 GitHub 的 Security Advisory 功能私密报告。

### 响应时间

| 阶段     | 时间       |
| -------- | ---------- |
| 确认收到 | 24 小时内  |
| 初步评估 | 3 个工作日 |
| 修复计划 | 7 个工作日 |
| 发布补丁 | 14-30 天   |

## 已实施的安全措施

### 认证与授权

- JWT + HTTP-Only Cookie 会话管理
- bcrypt 12 rounds 密码哈希
- 超级管理员独立认证体系
- API Key 认证（SHA-256 哈希存储，`ak_` 前缀 + 32 字节随机）
- 细粒度权限控制（8 种权限 + 通配符）

### 输入验证

- Zod 运行时 schema 验证覆盖所有 API 端点
- 文件上传白名单验证和大小限制
- SQL 注入防护（Prisma ORM 参数化查询）

### 访问控制

- 多租户数据隔离（所有查询包含 `tenantId`）
- 速率限制防暴力破解
- CSRF 防护
- 中间件统一鉴权

### 基础设施

- Standalone 输出模式，最小化攻击面
- 环境变量验证，防止配置错误
- 结构化日志记录所有安全事件
- 告警系统监控异常行为

## 部署安全清单

- [ ] 使用强 JWT_SECRET（32+ 字符随机字符串）
- [ ] 启用 HTTPS
- [ ] 数据库不暴露到公网
- [ ] 设置防火墙规则
- [ ] 启用日志和监控
- [ ] 定期备份数据库
- [ ] 定期更新依赖
